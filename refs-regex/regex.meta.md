1. 이스케이프

"정규 표현식 - 문자 검색" 에서도 이스케이프에 대해서 짧게 설명했지만,
앞으로 점점 더 많은 메타 문자들을 만나게 될 터라 한 번 더 이스케이프에 대해 정리를 하고 넘어가야 할 듯...

메타 문자는 정규 표현식안에서 특별한 의미를 가진다.
.(마침표)는 뉴라인 문자를 제외한 아무 문자와 일치할 수 있다.
[]는 문자 집합을 의미한다.
따라서, 마침표나 열고 닫는 대괄호를 문자로써 검색하려면 \(역슬래쉬)로 이스케이프를 해 줘야만 한다.

[예문]
var myArray = new Array();
...
if (myArray[0] == 0)
{
   ...
}

[정규 표현식]
myArray[0]

[결과]
var myArray = new Array();
...
if (myArray[0] == 0)
{
   ...
}

위 예제에서는 "myArray[0]"을 찾으려 했지만, 일치하는 결과가 하나도 없다.
[0]이 문자 집합으로 인식되어, "myArray0" 을 찾으려 했기 때문이다.

여는 대괄호나 닫는 대괄호를 문자 그대로 인식되게 하려면 아래와 같이 이스케이프를 해줘야 한다.

[정규 표현식]
myArray\[0\]

만약 배열의 0~9번째까지에 해당하는 문자열을 검색하려면 아래와 같이 정규 표현식을 작성해야 한다.

[정규 표현식]
myArray\[[0-9]\]

문자 "["와 "]"를 인식시키기 위해 이스케이프를 하였고, 0~9사이의 숫자를 검색하기 위해 다시 문자 집합([])을 사용하였다.

지금까지 배운 메타 문자가 .(마침표)와 [](문자 집합)뿐이지만, 모든 메타 문자는 \(역슬래쉬)로 이스케이프 시킬 수 있다.
또한, \(역슬래쉬) 또한 메타 문자라 하였다. 따라서, \\로 이스케이프 시켜야 "\"를 문자로 인식시킬 수 있다.


2. 공백 메타 문자

멀티라인 문자열에는 눈에 드러나진 않지만, 다양한 유형의 공백 문자가 포함되어 있다.
이들을 찾을 수 있는 공백 메타 문자들로는 다음과 같은 것들이 존재한다.
[\b] : 백스페이스 (잘 안 쓴다)
\f : form feed
\n : new line
\r : carriage return
\t : horizontal tab (일반적인 탭)
\v : vertial tab (잘 안 쓴다)

뭐, 기본적인 프로그래밍 지식이 있다면 위 녀석들은 이미 익숙할 것이다.
공백 메타 문자처럼, \(역슬래쉬)와 함께 사용하여야만 메타 문자가 되는 녀석들도 존재한다.

아래 예제를 통해 공백 메타 문자의 사용법에 대한 감을 잡아보자.
CSV 데이터들 중 불필요한 공백 라인을 찾아내는 예제이다.

[예문]
"101", "AAA", "BBB"
"102", "CCC", "DDD"

"103", "EEE", "FFF"

[정규 표현식]
\r\n

[결과]
"101", "AAA", "BBB"
"102", "CCC", "DDD"..............................................
.......................................................................
"103", "EEE", "FFF"

만약, 윈도우 시스템이 아닌, 유닉스/리눅스 시스템이었다면, "\n"으로만 검색해도 될 것이다.
따라서, 플랫폼을 가리지 않는 정규 표현식을 작성하려면, "\n"은 필수로, "\r"은 선택적으로 검색할 수 있어야 할 것이다.
이 부분에 대해선 "정규 표현식 - 수량자" 문서의 "문자가 없거나 하나만 있는 경우 찾기" 챕터에서 살펴볼 수 있다.


3. 클래스 메타 문자

문자 집합은 검색에서 쓰는 가장 흔한 형태이다.
편리성과 명확성을 위해, 자주 사용하는 문자 집합들의 타입을 특수한 메타 문자들로 표현할 수 있다.

이렇게 타입별 문자 집합을 인식되게 해주는 메타 문자들을 클래스 메타 문자라고 한다.

1) 숫자

정규 표현식 [0123456789]는 범위 지정자를 활용하여, [0-9]로 줄여 표현할 수 있다.
그리고, [0-9]를 다시 "\d"로 대신 표현할 수 있다.
또한, 숫자를 제외하려 할 때 [^0-9]로 표현하는데, 이를 "\D"로 대신 표현할 수 있다.
\d : [0-9]
\D : [^0-9]

앞서 사용했던 예제를 한번 더 재활용해 보자.

[예문]
var myArray = new Array();
...
if (myArray[0] == 0)
{
   ...
}

[정규 표현식]
myArray\[\d\]

여는 대문자([)와 닫는 대문자(])를 위해 이스케이프를 하였고, [0-9]를 대체하기 위해 \d를 사용하였다.

정규 표현식의 모든 문법은 대소문자를 구별한다.
\d는 숫자 하나와 일치하고, \D는 그 반대의 의미이다.

앞으로 살펴볼 클래스 메타 문자들도 소문자와 대문자는 서로 반대의 의미를 가진다.

2) 영문자, 숫자, _(언더바)

숫자 집합 외 자주 사용하는 문자 집합으로는 영문자(대소문자), 숫자, _(언더바)를 포함하는 집합이 있다.
보통 e-mail 주소, 웹 URL 등이 이 집합으로 모두 검색이 가능하다.
\w : [A-Za-z0-9_]
\W : [^A-Za-z0-9_]

영숫자, 숫자, 영숫자, 숫자, 영숫자, 숫자 6글자의 조합을 요구하는 예제를 살펴보자.

[예문]
A1C2E3
1A1B3Z
B4AAX9
112233

[정규 표현식]
\w\d\w\d\w\d

[결과]
A1C2E3
1A1B3Z
B4AAX9
112233

3) 공백

모든 공백 문자를 포함하는 클래스 메타 문자는 다음과 같다.
\s : [\f\n\r\t\v]
\S : [^\f\n\r\t\v]

분명 모든 공백 문자를 포함한다 하였는데, [\b](백스페이스)는 빠져 있다. (왜 그럴까?)

모든 타입의 공백을 제외하고 첫번째 글자를 일치시켜 보자.

[예문]
 a
	a

[정규 표현식]
\S

[결과]
 a
	a

공백 문자를 찾을 경우에는 다음과 같이 작성하면 된다.

[정규 표현식]
\s

[결과]
.a
....a


4. 8/16 진수 표현하기

자주 사용되진 않지만, 그래도 알아두도록 하자.

1) 8진수 표현하기

8진수(octal) 값은 두 자리나 세 자리 정수 앞에 \O(영문자 O)를 붙여 표시한다.
\O11은 아스키 문자 9에 해당하며, 아스키 문자 9는 "\t" 탭을 의미한다.

2) 16진수 표현하기

16진수(hexadecimal) 값은 \x를 붙여 표시한다.
\x0A는 아스키 문자 10에 해당하며, 이는 "\n" 뉴라인 문자를 의미한다.
